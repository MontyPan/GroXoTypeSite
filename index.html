What is GroXoType?

GroXoType is a GXT prototyping webapp based on GXT. It is a WYSIWYG editor in which users can design layout without writing codes and see the UI without waiting for compilation. Furthermore, GroXoType generates equivalent ui.xml code in most situations.

Info

GroXoType does not provide a DnD editor, user must know meaning of parameters of GXT layout containers such like HorizontalLayoutData.

How to use?

  • Add the GroXoType source code to the project build path.

    Notice

    Sorry, GroXoType does not support maven repo yet. You need to set the source code / build path on your hand. :(
  • Insert the GroXoType module into your project gwt.xml. 
    <inherits name="montyPan.groxotype.GroXoType" />
  • Write the EntryPoint class: 
    	@Override
	public void onModuleLoad() {
		ProviderUtil.addAll();
				
		Viewport vp = new Viewport();
		vp.add(new AddButton(vp));
		RootPanel.get().add(vp);
	}
  • Browse the host page, you will find a button filled hole browser. Click it and choose which component you want to add.
  • When you finish layout process, right click and bring a context menu. Then click the "Generator" button, you will get the ui.xml code.

Notice

If you want to use the code generator function, only one AddButton can be used in initial layout.

Info

AddButton is a normal GXT component. You can add it on any container in your exist project. :)

Write Provider

If you has a customize widget which name is Foo, want Foo can be used in GroXoType. All you need to do is write a ComponentProvider

Basic Concept

  • Create a class, FooProvider in this case, extends ComponentProvider
  • Implement these methods:
    • getCategory(): The category of Foo.
    • buttonText(): The name of Foo.
    • genComponent(): This method will return an actual Foo instance.
  • If you need parameters when genComponent() is invoked, override genSettingView() and make it to return a setting widget.
Before AddButton initialized, make sure ComponentSelector.addProvider() was invoked like this: 
ComponentSelector.addProvider(new FooProvider());

Code Generator

Notice

GroXoType can't generate code of all kind of widgets without rewrite core of GroXoType's generator. If you follow these guildlines but still can't generate correct code (like TabPanel), don't be sad. Because it could be the limitation of GroXoType, not your fault. I am still finding better generate alogrithm :'(

If Foo has some attributes that will apear in ui.xml, you need to override genAttrMap() in FooProvider. The return value of genAttrMap() is a HashMap<String, Object>, make sure the toString() behavior of the value you put in HashMap is expected. Generator will call toString() to generate ui.xml code.

If Foo has child widget wrapped in pseudo-element, <f:widget> of FieldLabel for example, you need to override genContext(). The return value of genContext() is a ArrayList<String>, each element is a line of code will be generated. Use the constant GenUtil.CHILD in string to represent child widget, generator will replace GenUtil.CHILD to real code of child widget.
