What is GroXoType?

GroXoType is a GXT prototyping webapp based on GXT. It is a WYSIWYG editor in which users can design your layout without writing codes and see the UI without waiting for compilation. Furthermore, GroXoType generates equivalent ui.xml code in most situations.

Tips

GroXoType does not provide DnD feature. Users must understand the meaning of GXT LayoutData's parameters, such as HorizontalLayoutData.

How to use?

  • Add the GroXoType source code to the project Build Path.

    Notice

    GroXoType does not support maven repo at this stage. Users will need to set the source code / build path by themselves.
  • Insert the GroXoType module into the project's gwt.xml. 
    <inherits name="montyPan.groxotype.GroXoType" />
  • Write the EntryPoint class: 
    	@Override
	public void onModuleLoad() {
		ProviderUtil.addAll();
				
		Viewport vp = new Viewport();
		vp.add(new AddButton(vp));
		RootPanel.get().add(vp);
	}
  • Browse the host page, users will find the window filled with one single button. Click it and choose which component you want to add.
  • When the prototyping is completed, right click and a context menu will appear. Click the "Generator" button, and you will get the ui.xml code.

Notice

If users intend to use the code generator function, only one AddButton can be added on initial layout.

Tips

AddButton is a normal GXT component without any restriction. You can add it on any container in your existing project. :)

Write Provider

If you have a customize widget named Foo, and you want it to be used in GroXoType. All you need to do is write a ComponentProvider

Basic Concept

  • Create a class, FooProvider in this case, which extends ComponentProvider
  • Implement these methods:
    • getCategory(): The category of Foo.
    • buttonText(): The name of Foo.
    • genComponent(): This method will return an actual Foo instance.
  • If you need parameters when genComponent() is invoked, override genSettingView() and make it to return a setting widget.
Before AddButton is initialized, make sure ComponentSelector.addProvider() was invoked like this: 
ComponentSelector.addProvider(new FooProvider());

Code Generator

Notice

GroXoType cannot generate code of all kind of widgets without rewriting core of GroXoType's generator. If you follow these guildlines but still can't generate correct code, don't be frustrated. Because it could be due to the limitation of GroXoType. We are still working on better generate alogrithm.

If Foo has some attributes that will apear in ui.xml, you need to override genAttrMap() in FooProvider. The return value of genAttrMap() is a HashMap<String, Object>, make sure the toString() behavior of the value you put in HashMap is expected. GroXoType generator will call toString() to generate ui.xml code.

If Foo has child widget wrapped in pseudo-element, <f:widget> of FieldLabel for instance, you will need to override genContext(). The return value of genContext() is a ArrayList<String>, each element is a line of code that will be generated. Use the constant GenUtil.CHILD in string to represent child widget, generator will replace GenUtil.CHILD with real code of child widget.
